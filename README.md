# 🍕 PizzaToPizza Frontend

## Загальна інформація про проект

PizzaToPizza Frontend є клієнтською частиною дипломного full-stack вебпроєкту, створеного як сучасний сервіс онлайн-замовлення піци. Основною метою проекту було розробити повноцінний вебзастосунок, який імітує реальну систему доставки їжі та демонструє практичні навички побудови комерційного SPA-додатку з використанням React, REST API та JWT авторизації.

Проєкт орієнтований на дві категорії користувачів: звичайних клієнтів сервісу та адміністраторів системи. Клієнт може переглядати асортимент піц, сортувати товари, використовувати промокоди, оформлювати замовлення та відстежувати їх статус. Адміністратор має окремий функціонал керування каталогом товарів, категоріями, акціями та всіма замовленнями користувачів.

Проєкт реалізовано у форматі Single Page Application, що забезпечує швидку навігацію між сторінками без перезавантаження браузера, сучасний UX та високу швидкість роботи інтерфейсу.

## Основні цілі створення проекту

Під час розробки були поставлені наступні задачі:

- створити сучасний frontend на React;
- реалізувати підключення до backend API;
- організувати авторизацію через JWT Token;
- реалізувати ролі користувачів;
- створити кошик покупок;
- реалізувати систему замовлень;
- додати систему промокодів;
- реалізувати рейтинг товарів;
- створити адміністративну панель;
- забезпечити адаптивність під мобільні пристрої;
- використати актуальні практики розробки SPA-застосунків.

---

## Використані технології

Frontend частина побудована на сучасному стеку JavaScript технологій:

- **React** — бібліотека для побудови інтерфейсу користувача;
- **Vite** — сучасний bundler та dev server;
- **React Router DOM** — маршрутизація між сторінками;
- **Axios** — HTTP клієнт для роботи з backend API;
- **React Toastify** — красиві toast-повідомлення;
- **CSS3** — кастомна стилізація;
- **Bootstrap classes** — частково використані кнопки та допоміжні стилі;
- **LocalStorage** — збереження токена та промокодів;
- **JWT** — система авторизації.

## Авторизація

JWT авторизація працює за схемою:

1. Користувач вводить email та пароль
2. Backend повертає token
3. Дані зберігаються у `localStorage`
4. `AuthContext` оновлюється
5. Відкривається доступ до приватних сторінок

### Logout

- token видаляється
- user очищується
- приватний доступ блокується
- header оновлюється

## LocalStorage
